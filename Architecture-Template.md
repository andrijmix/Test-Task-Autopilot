# Шаблон архітектурного документа

## 1. Контекст

### 1.1 Проблема
- Потрібно автоматизувати місію коптера: зліт з точки A, переліт до точки B, утримання висоти 100 м, точна посадка в B.
- Ключова технічна умова: керування в режимі Stabilize через RC Override.

### 1.2 Вхідні дані
- Точка A: [50.450739, 30.461242]
- Точка B: [50.443326, 30.448078]
- Цільова висота: 100 м
- Інструменти: Mission Planner, DroneKit

### 1.3 Цілі системи
- Безпечний автономний прохід місії A->B.
- Висока точність посадки у B.
- Відмовостійкість і кероване завершення місії.

### 1.4 Не-цілі
- Побудова повного GCS.
- Складна 3D траєкторна оптимізація.

## 2. Вимоги

### 2.1 Функціональні
- Підключення до автопілота та валідація pre-flight стану.
- ARM + takeoff до 100 м.
- Навігація до B в Stabilize через RC Override.
- Утримання висоти на крейсері.
- Фінальний підхід і посадка в B.
- Логування телеметрії, команд, переходів станів.

### 2.2 Нефункціональні
- Безпечне обмеження RC команд (PWM bounds, rate limit).
- Таймаути та fail-safe для ключових станів.
- Конфігурованість параметрів без зміни коду.

### 2.3 Критерії приймання
- Висота: ___ м +/- ___ м.
- Похибка посадки: <= ___ м.
- Успішне завершення місії в SITL: ___/___ прогонів.

## 3. Архітектура високого рівня

### 3.1 Компоненти
- Mission Controller (FSM)
- Navigation Service (distance/bearing/heading)
- Altitude Controller (throttle loop)
- Horizontal Controller (yaw + pitch/roll)
- RC Override Adapter (канали + safety)
- Telemetry & Logger
- Fail-safe Manager

### 3.2 Взаємодія компонентів
1. Mission Controller читає телеметрію.
2. Навігація обчислює помилки позиції/курсу.
3. Контролери формують RC команди.
4. RC Adapter застосовує обмеження і відправляє override.
5. Logger фіксує стани, помилки, команди.
6. Fail-safe Manager може форсувати ABORT.

## 4. Станова модель (FSM)

- INIT
- ARM_TAKEOFF
- CRUISE_TO_B
- APPROACH_FINE
- LANDING
- COMPLETE
- ABORT

Для кожного стану вказати:
- Entry умови
- Exit умови
- Таймаут
- Помилки/виключення
- Дію при timeout

## 5. Алгоритми керування

### 5.1 Геодезія
- Формули distance та bearing.
- Нормалізація кутів у діапазоні [-180, 180].

### 5.2 Контур висоти
- Вхід: помилка висоти = h_target - h_current.
- Вихід: throttle PWM.
- Налаштування: Kp, Ki, Kd (за потреби), anti-windup.

### 5.3 Контур горизонтального руху
- Вхід: bearing error, distance to target.
- Вихід: yaw/pitch/roll PWM.
- Двофазна логіка: cruise/fine.

### 5.4 Фінальний підхід
- Перехід у fine при distance < R_fine.
- Зниження агресивності команд ближче до B.
- Старт посадки при distance < R_land і speed < V_land.

## 6. Безпека

- Обмеження PWM: min/max.
- Rate limiter на зміну команд.
- Deadband біля нульової помилки.
- Watchdog на скидання override.
- Геозона/максимальний радіус від маршруту.
