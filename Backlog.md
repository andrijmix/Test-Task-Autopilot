# Технічний Backlog: Autopilot A->B (Stabilize + RC Override)

## 1) Мета і критерії приймання

- Мета: автоматизований зліт з точки A, переліт у точку B, утримання висоти 100 м, точна посадка в B.
- Обмеження: режим польоту Stabilize, керування через RC Override.
- Джерела: Mission Planner, DroneKit.

### Acceptance Criteria (мінімум)

- Коптер досягає району точки B.
- У крейсері висота утримується в межах +/- 3 м від 100 м.
- Після входу у фінальний радіус знижується та сідає без аварійної поведінки.
- Усі ключові події логуються (стани FSM, координати, висота, RC команди, помилки).

### Acceptance Criteria (високий бал)

- Похибка посадки <= 3 м від B (ціль <= 1.5 м).
- Відсутні перерегулювання/розгойдування на фінальному етапі.
- Оброблено базові fail-safe сценарії (timeout, втрата GPS, втрата телеметрії).

## 2) План робіт (епіки -> задачі)

## Epic A. Setup і середовище

### A1. Ініціалізація проєкту
- Результат: структура проєкту, конфіг, точки A/B, висота, пороги.
- DoD:
  - Є файл конфігурації (JSON/YAML/Python dataclass).
  - Координати і пороги не зашиті всередині логіки контролера.
- Оцінка: 2-3 год.

### A2. Залежності й запуск
- Результат: відтворюваний запуск скрипту.
- DoD:
  - Встановлення `dronekit` описано в README.
  - Команда запуску і параметр `--connect` задокументовані.
- Оцінка: 1-2 год.

### A3. Базова телеметрія
- Результат: стабільне підключення до Vehicle.
- DoD:
  - Перевірка heartbeat, armable, mode, GPS fix.
  - Лог стартових параметрів.
- Оцінка: 2-3 год.

## Epic B. Ядро керування польотом

### B1. FSM (state machine)
- Стани: INIT, ARM_TAKEOFF, CRUISE_TO_B, APPROACH_FINE, LANDING, COMPLETE, ABORT.
- DoD:
  - Явні переходи, причини переходів у логах.
  - Таймаут на кожен стан.
- Оцінка: 3-4 год.

### B2. Гео-модуль
- Результат: distance, bearing, нормалізація кутів.
- DoD:
  - Покрито unit-тестами.
  - Похибка обчислень у допустимих межах.
- Оцінка: 2-3 год.

### B3. RC Override абстракція
- Результат: безпечне встановлення каналів RC.
- DoD:
  - Saturation меж (min/max pwm).
  - Центрування каналів після кроку керування.
  - Watchdog для скидання override при аварії.
- Оцінка: 3-5 год.

### B4. Контур висоти (throttle)
- Результат: утримання 100 м у Stabilize через throttle channel.
- DoD:
  - Працює P/PI регулятор висоти.
  - Anti-windup (якщо PI).
  - Лог висотної помилки і команди throttle.
- Оцінка: 4-6 год.

### B5. Контур курсу/напряму (yaw + pitch/roll)
- Результат: рух у напрямку B.
- DoD:
  - Коптер орієнтується на bearing до B.
  - Горизонтальна помилка зменшується монотонно в середньому.
- Оцінка: 4-6 год.

## Epic C. Точний підхід і посадка

### C1. Двофазне зближення
- Фаза 1 (cruise): активніші RC команди.
- Фаза 2 (fine): приглушені команди ближче до B.
- DoD:
  - Порог радіусу переходу (наприклад 50 м) параметризовано.
  - Немає різких стрибків команд при перемиканні фаз.
- Оцінка: 3-4 год.

### C2. Фінальне вирівнювання над B
- Результат: мінімальна горизонтальна швидкість перед зниженням.
- DoD:
  - Умова початку посадки: дистанція < R_land, швидкість < V_land.
  - Порогові значення в конфігу.
- Оцінка: 3-5 год.

### C3. Посадка та завершення
- Результат: контрольоване зниження, зупинка мотора/дизарм.
- DoD:
  - Визначення touchdown за висотою/верт. швидкістю/станом.
  - Безпечний вихід з override.
- Оцінка: 2-4 год.

## Epic D. Надійність і безпека

### D1. Fail-safe сценарії
- Сценарії: втрата GPS, timeout стану, втрата лінку, вихід за геозону.
- DoD:
  - Для кожного сценарію є реакція ABORT.
  - Реакція задокументована.
- Оцінка: 4-6 год.

### D2. Обмеження команд
- Результат: захист від небезпечних RC імпульсів.
- DoD:
  - Границі PWM, rate limiter, deadband.
  - Логування кліпінгу команд.
- Оцінка: 2-3 год.

## Epic E. Тестування

### E1. Unit тести
- Покриття: гео-функції, нормалізація кутів, мапінг помилки в PWM.
- DoD:
  - Критичні обчислення покрито тестами.
- Оцінка: 3-4 год.

### E2. SITL сценарії
- Нормальний сценарій.
- Боковий вітер.
- Шум GPS.
- Телеметричний timeout.
- DoD:
  - По кожному сценарію є лог і підсумок.
- Оцінка: 5-8 год.

### E3. Критерії точності
- Результат: вимірювання похибки посадки.
- DoD:
  - Скрипт/метод підрахунку miss distance (м).
  - Звіт з мін/макс/середнім.
- Оцінка: 2-3 год.

## Epic F. Документація і демонстрація

### F1. README (операційний)
- Параметри запуску, безпечний чекліст перед стартом, типові помилки.
- Оцінка: 2-3 год.

### F2. Mission report
- Траєкторія, графік висоти, похибка посадки, висновки.
- Оцінка: 2-4 год.

## 3) Пріоритети і послідовність

1. A1 -> A2 -> A3
2. B1 -> B2 -> B3
3. B4 + B5
4. C1 -> C2 -> C3
5. D1 + D2
6. E1 -> E2 -> E3
7. F1 -> F2

## 4) Релізний план

### Milestone M1 (MVP)
- A + B1/B2/B3 + базові B4/B5.
- Результат: зліт, доліт до B, груба посадка.

### Milestone M2 (стабільність)
- C1/C2/C3 + D1/D2.
- Результат: безпечна стабільна місія.

### Milestone M3 (точність)
- Тюнінг регуляторів, E2/E3.
- Результат: максимальне наближення до «ідеально в B».

## 5) Ризики

- Stabilize + RC Override сильно залежить від тюнінгу PID і умов середовища.
- GPS похибка може домінувати над алгоритмом на фіналі.
- Різні прошивки/параметри ArduPilot впливають на реакцію.

## 6) Мінімальний Definition of Done проєкту

- Скрипт проходить SITL місію A->B із посадкою.
- Висота в крейсері тримається близько 100 м.
- Похибка посадки виміряна та задокументована.
- Є fail-safe і інструкція запуску.
